# 玄枢产品化重构方案

## 目标

把当前站点重构为面向中国大陆用户的成熟八字工具网站。用户进入 3 秒内应知道：这是免费八字排盘工具、能生成哪些内容、下一步在哪里开始。

## 信息架构

1. 顶部导航：产品能力、使用流程、报告内容、开始排盘、常见问题。
2. 首页 Hero：品牌定位、核心价值、双操作按钮、信任说明、真实报告预览。
3. 产品价值：结构化排盘、五行可视化、关系解读、阶段节奏。
4. 使用流程：填写出生信息、生成命盘、阅读报告、保存分享。
5. 报告章节：总览、四柱、五行、十神、地支关系、大运、流年、行动建议。
6. 开始排盘：注册信息与出生信息合并为一次提交；已有账号可直接登录。
7. 结果预览：展示最终报告的信息密度与移动端截图效果。
8. 常见问题：准确性、隐私、出生时间未知、使用边界。
9. 页脚：隐私、条款、产品边界和联系方式预留。

## 首页版式

- 桌面端 Hero 采用左侧文案、右侧报告预览的 46/54 布局。
- 主题图只承担东方仪轨氛围，不承载页面文字或按钮。
- 首屏底部露出下一段内容，避免海报式封闭首屏。
- 移动端先展示标题和主操作，再展示压缩后的报告预览。

## 设计 Tokens

- 背景：`#0f1114`、`#15181d`。
- 主文字：`#f3ede3`；辅助文字：`#cbbfb0`；弱文字：`#958e8e`。
- 主操作：`#c25344`；铜金强调：`#d8b48e`；矿物青：`#5d7c92`。
- 圆角：10 / 16 / 24 / 32 px，用于不同层级，功能卡不超过 16 px。
- 标题使用中文宋体栈，正文使用中国大陆常见黑体栈，数据使用等宽字体。
- 间距采用 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 px。

## 核心组件

- `SiteHeader`：桌面锚点导航和移动端折叠菜单。
- `HeroSection`：定位、操作、信任点和命盘预览。
- `ValueSection`、`HowItWorks`、`ReportChapters`：完整产品叙事。
- `OnboardingForm`：注册与出生信息一次提交，已有账号登录。
- `ReportPreview`：展示报告成果，不使用空白占位。
- `FAQSection`：原生可访问折叠内容。
- `ResultSection`：章节导航、报告封面、数据图表、解释与分享操作。

## 动效

- 首屏内容 0.6 秒淡入上移。
- 报告预览 10 秒以内轻微上下浮动。
- 章节进入视口时分组显现。
- 五行条从左向右加载。
- 按钮 hover 提亮，按下轻微缩放。
- 全部动画遵守 `prefers-reduced-motion`，移动端降低位移。

## 中文文案

- 不使用“实验室、命盘仪、校准、模块、初始化”等开发者或设定化语言。
- 页面用词固定为：立即开始排盘、查看报告内容、重新填写、保存为图片、生成分享文案。
- 所有说明采用中国大陆简体中文，短标题、短段落、明确下一步。
- 不做绝对预测，不使用恐吓表达，保留传统文化研究与娱乐参考边界。

## 移动端

- 375 / 390 / 430 px 均为一列布局，主要触控区不低于 44 px。
- Hero 预览移到文案下方；表单标签在上、控件在下。
- 结果章节一屏一张重点卡；复杂表格在卡内横向滚动。
- 分享操作固定为清晰的按钮组，不依赖小图标。

## 本阶段验收

- 首页不再是主题图铺满的海报。
- 从首页到提交出生信息再到报告的路径连续。
- 登录前能看到完整产品价值和报告内容。
- 登录后结果不是表格集合，而是可阅读、可截图的章节化报告。
- 中文自然、移动端无页面级横向溢出、动画可减弱。
